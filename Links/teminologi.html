Terminologi

Hem Versionshantering

Repository betyder: Det centrala förvaret där mina kolleger kan te.x FORK:a eller kopiera min Repository där alla kan arbeta ifrån. du använder din Repository till din klient te,x.

Branch betyder: skapar en tidslinje med alla dina Commit (ändringar) i din programmering, det vill säga att om du gör Commits och du har ett projekt med 5 andra personer så kan man enkelt följa vad du har gjort genom arbetet gång genom Branch:n och branchen går oftas till en master eller en main Branch.

Commit & Push betyder: En ändring i koden har skett och då märker min klient Git Extensions det, nu när ändringen är gjort så går jag in på mitt programm och kollar så allt stämmer och mergar sen commitar och sedan pushar jag upp det till mitt Repository. Commit betyder ändring kan man säga och push betyder att man skickar ändringen till sitt arbetes Repository.

Merge betyder: Är den punkt på baslinjen där uppdateringar gjorda i en branch sparas i huvudprojektet.

Pull request betyder: Pull request är föreslagna ändringar av ett arkiv som skickats av en användare och accepterat eller avvisat av ett arkivs samarbetspartners. Liksom frågor har pull request var och en sitt eget diskussionsforum.

Tags: är versionshanteringsprogrammens sätt att strukturera förändringarna gjorda i projektet. Inga förändringar görs i en tag, istället ska en branch bildas utifrån en tag.

Trunk: ett bregrepp som betyder huvudspår som i huvudsak förekommer i subversion. mest använder man Master i Git

Cherry picking:git cherry-pick är ett användbart kommando som gör att arbitrary Git-commits kan väljas genom referens och läggas till det aktuella arbetande i HEAD. Cherry picking är att plocka en commit från en gren och tillämpa den på en annan.cherry-pick kan vara användbart för att ångra ändringar.till exempel att en commit har gjorts av misstag till fel gren så kan du byta till rätt gren och välja commit där den ska höra hemma.
