Versionshantering

Hem Grundläggande terminologi

Versionshantering betyder version av källkodsprogram som kan används smart och smartare. Te.x i denna kursen har vi lärt oss använda Github och Git Extensions där man kan arbeta tillsammans med sina kollegor , smidigt att sammarbeta och få ett struktuerat arbete.
I versionshantering kan du se när ett objekt eller en fil ändrades och vem som ändrade det. Du kan också se när egenskaper (information om filen) har ändrats. Om någon till exempel ändrar förfallodatumet för ett listobjekt visas den informationen i versionshistoriken. Du kan också se kommentarer som andra gör när de checkar in filer i bibliotek. Om du har gjort ett misstag i en aktuell version, om den aktuella versionen är skadad eller om du bara gillar en tidigare version bättre, kan du ersätta den aktuella versionen med en tidigare version. Den återställda versionen blir den nya aktuella versionen. Finns många olika program men har läst och förstått mig på att Git är det som används mest i programmeringsvärlden.

Fördelar med att använda ett Versionshantering-system te.x Git Extensions är att man kan vara många på ett projekt samtidigt och man är synkade med alla genom Klienten de vill säga i detta fallet Git Extensions. Att kunna följa folks arbete genom ett projekt gång är en stor fördel om något skulle gå snett, då man kan gå tillbaka och se vad och vem har gjort "felet". Till exempel vet alla till viss del vad alla andra gör i projektet. Administratörer har detaljerad kontroll över vem som kan göra vad, och att hantera ett centralt system är mycket enklare än att hantera en lokal databas på varje klient. En liten sammanfattning skapa eller klona ett förvar, göra ändringar, preparera och spara ändrigarna samt visa historiken av alla ändrigana som sparats i förvaret. Nackdel om att använda Versionshantering-system är att man måste vara fler än en person tänker jag. Nackdel att inte använda en VCS är att man inte får en back-up och där förlitar sig på det lokala förvaret. Att arbeta på samma område i ett projektet, så kan det uppstå en konflikt mellan två partner vilket kräver mer tid än vad som hade behövs.
